# 第三阶段：新功能扩展与UI优化 (2023)

## 目标
响应用户需求，增强系统的可用性和界面美观度，主要包含三个新增核心功能和界面调整：
1. **历史纪要管理（保存/打开/重命名）**
2. **自定义保存路径设置**
3. **音视频媒体预览功能**

## 实施细节

### 1. 全局配置与保存功能 (src/config.py)
引入了一个简单的本地持久化机制：
- 使用 `settings.json` 保存用户设置。
- 增加 `get_save_dir()` 和 `set_save_dir()` 方法。
- 系统默认在工程根目录下创建一个 `saves` 文件夹。
- ASR+LLM 流程处理完毕后，系统自动以 `原文件名_纪要_当前时间.md` 为名称保存至目标文件夹。

### 2. UI 界面解构与重构 (src/ui/main_window.py)
窗体整体大小提升至 `1280x850`，布局进行了彻底划分：
- **左侧边栏 (Sidebar)**: 
  - 放置 `QListWidget` 展示 `saves/` 目录下的 `.md` 文件列表（按修改时间倒序）。
  - 集成了自定义右键菜单 (Context Menu)，支持直接在列表中进行 **重命名** 和 **删除** 操作。
  - 点击列表项可将历史文件内容加载至右侧区域。
  - 侧边栏顶部新增 `设置` 按钮。
- **右侧工作区 (Main Content)**:
  - 顶部维持工具栏：选择文件、展示文件路径、开始生成。
  - 引入了 `QTabWidget`，将功能分为 **"总结与纪要"** 和 **"媒体预览"** 两个页面。

### 3. 子功能模块

#### 设置弹窗 (src/ui/settings_dialog.py)
设计了一个 `QDialog` 供用户更改纪要默认的存放路径，使用 `QFileDialog.getExistingDirectory` 实现文件夹选取。

#### 媒体预览 (Tab 2)
调用 `PyQt6.QtMultimedia` 实现基础播放器：
- `QVideoWidget` 用作视频渲染区域。
- `QMediaPlayer` 作为控制核心，通过信号 `positionChanged` 与 `durationChanged` 驱动UI。
- 引入基础播放控制条组件：播放/暂停 `QPushButton` 和用于进度快进后退的 `QSlider`，以及时间标签。
- 当用户通过顶部工具栏选择文件后，播放器会自动加载资源的本地 `QUrl` 等待播放。

## 当前状态
- 新增功能均已落地完毕代码实现。
- 继续保持了此前的后端 LLM 生成流程和音频处理逻辑。
- 系统更加趋于产品级的可用状态，拥有了历史查询和原生多媒体能力。
